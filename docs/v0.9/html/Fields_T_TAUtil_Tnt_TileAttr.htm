The TileAttr type exposes the following members.

Fields
  NameDescription
AttrLength
The length of an attribute structure in bytes in a TNT file.
Feature

Offset in the feature array of the feature located in this square.

0xFFFF if none, 0xFFFC (-4) if void. I've also seen 0xFFFE (-2) on some of the early Cavedog maps such as Lava Run and AC02, but have no idea what it means. Please contact me if you have any information.

FeatureNone
The feature value representing no feature.
FeatureUnknown
Feature value present in some maps. Its purpose is unknown.
FeatureVoid
The feature value representing a void cell.
Height
The height at this point.
Pad1
No known purpose (my personal guess is that it is padding to get to 4 bytes).
SctVersion2AttrLength
The length of an attribute structure in bytes in a version 2 SCT file.
SctVersion3AttrLength
The length of an attribute structure in bytes in a version 3 SCT file.
